**Key Technical Concepts**
-   **Static Data Fallback**: A robust pattern was used to make the UI resilient. If an API call fails or returns empty data, the component falls back to a locally stored static mock data set. This was key to fixing production UI bugs.
-   **State-Driven Workflow**: A  field ('pending', 'verified', etc.) was added to the  model to create a verification workflow between cooperatives (web) and field agents (mobile).
-   **Role-Based UI Rendering**: React components conditionally render UI elements based on the authenticated user's type (), creating different experiences for buyers, sellers, and unauthenticated users.
-   **QR Code Data Transfer**: A flow was implemented where complex JSON data is Base64-encoded by the backend into a QR code, which is then scanned and decoded by the mobile app to pre-fill forms and navigate to specific screens.
